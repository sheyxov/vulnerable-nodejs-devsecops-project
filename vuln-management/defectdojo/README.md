This directory documents the **vulnerability management integration**
used in the DevSecOps pipeline.

The project uses **DefectDojo** to centralize and manage security findings
generated by multiple security tools.

---

## Why DefectDojo?

DefectDojo provides:
- Centralized vulnerability tracking
- Tool-agnostic import support
- Severity normalization
- Visibility across the SDLC

This makes it suitable for managing findings from:
- SAST
- SCA
- Secret Scanning
- DAST

---

## Integration Overview

Security scan results are generated during the CI/CD pipeline and
automatically imported into DefectDojo.

The integration is implemented using a **custom Python script**
that interacts with the DefectDojo REST API.

---

## Supported Scan Types

The following scan results can be imported:
- Semgrep (SAST)
- Snyk (SCA)
- Gitleaks (Secret Scanning)
- OWASP ZAP (DAST)

---

## Configuration Model

All configuration is handled via **environment variables**.

No API keys, URLs, or credentials are stored in this repository.

Required variables include:
- DefectDojo URL
- API Token
- Product / Engagement identifiers

Refer to `upload_to_dd.example.py` for a configuration template.

---
